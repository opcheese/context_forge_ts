You don't need to build your context from scratch. Here are five templates you can copy, customize, and start using immediately.

## 1. The Developer Template

```
PERMANENT:
Role: [Your level] [language] developer
Style: [Your preferences: pure functions, OOP, etc.]
Testing: [Framework and approach]
Commits: [Convention]

STABLE:
Stack: [Framework + database + deployment]
Key files: [3-5 most important paths]
Patterns: [Architecture patterns you follow]

WORKING:
Task: [What you're building]
Constraints: [Libraries, time, compatibility]
Done when: [Acceptance criteria]
```

## 2. The Product Manager Template

```
PERMANENT:
Role: PM at [company type]
Audience: [Who reads your output]
Style: [Concise/detailed, data-driven/narrative]
Frameworks: [RICE, JTBD, etc.]

STABLE:
Product: [Name and one-line description]
Metrics: [Key metrics you track]
Current focus: [This quarter's OKR or goal]

WORKING:
Task: [PRD / analysis / update / brainstorm]
Input: [Data or docs to reference]
Output format: [Bullets, table, narrative, etc.]
```

## 3. The Technical Writer Template

```
PERMANENT:
Role: Technical writer
Voice: [Formal/casual, second-person/third-person]
Audience: [Developers, end-users, mixed]
Style guide: [Link or key rules]

STABLE:
Product: [What you're documenting]
Terminology: [Key terms and their correct usage]
Existing docs: [Where current docs live]

WORKING:
Task: [New page, update, API reference]
Scope: [What to cover, what to skip]
Length: [Target word count or section count]
```

## 4. The Researcher Template

```
PERMANENT:
Role: [Domain] researcher
Rigor: [Academic/applied/exploratory]
Bias awareness: Explicitly flag assumptions and limitations
Citation style: [Format preference]

STABLE:
Research area: [Topic and subtopic]
Key sources: [Foundational papers or datasets]
Current hypothesis: [What you're investigating]

WORKING:
Task: [Literature review, analysis, synthesis]
Input: [Papers, data, transcripts]
Output: [Summary, comparison table, critique]
```

## 5. The UX Designer Template

```
PERMANENT:
Role: UX/UI designer
Design system: [Tailwind, Material, custom]
Principles: [Accessibility-first, mobile-first, etc.]
Tools: [Figma, code-based, etc.]

STABLE:
Product: [What you're designing for]
Users: [Primary personas]
Constraints: [Platform, browser support, etc.]

WORKING:
Task: [New component, redesign, user flow]
Requirements: [Functionality and behavior]
Deliverable: [Wireframe, prototype, spec]
```
