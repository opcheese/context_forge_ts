### Feature 4: Persist brainstorm conversations — `49965cc`, `3243571`, `52ad702`, `6bd1b09`, `e5fa623`

**Files:** `src/hooks/useBrainstorm.ts`, `src/components/BrainstormDialog.tsx`, `src/components/BrainstormPanel.tsx`

Brainstorm messages were ephemeral — lost on close/refresh. Implemented localStorage persistence:

- Messages save immediately on every change (sync, not debounced — debounce caused data loss on close+F5)
- Load from localStorage on mount and session change
- LRU eviction caps at 5 stored sessions
- "Previous conversation restored (N messages)" banner on reopen
- Export to markdown button downloads full conversation
- Close button just closes (no more discard-only warning) — conversation survives
- Clear button explicitly wipes conversation + localStorage
- Browser `beforeunload` warning still active as safety net

**Key bug found during implementation:** Two `useEffect([sessionId])` hooks competed on mount — one cleared messages (`setMessages([])`), the other loaded from localStorage. The clear effect won the race and wiped stored data. Fixed by merging into a single effect.
